This is a tool for generation of SWATH ion library for quantification of SWATH-MS output in PeakView.
Back-End Operation:
  • Each query sequence is submitted independently to the backend for preprocessing.
  • Preprocessing:
    • The back-end would first looks for conflict in non-Ytype variable modifications and create multiple combination of the conflicted modifications. Each conflict solution scenario would be evaluate separately.
    • If there are Ytype modifications, each sequence passed above would have each Ytype modifications considered individually.
    • A static modification dictionary and a variable modification dictionary with information of all relevant modification are prepared for each iteration.
  • A different combination of variable modifications are generate for each variable modifications at non conflicted position.
    • If multiple patterns option was not selected and there are more than one non-conflicted positions for any variable modifications, the amount of total modifications site of this specific variable modifications would slowly decrease in linear fashion from the N to C terminus.
    • If multiple patterns option was selected, all possible patterns would be considered for non-conflicted positions.
    • If fill all positions option was selected, the back end would assume that all non-conflicted variable modifications would be assumed to be all filled or all absent.
    • The position of each modification are used to create the unique precursor sequence with decorated modification information.
  • For each combination generated of each iteration, the precursor sequence would be in-silico fragmented by b, y and/or Y ion series.
    • The mass of each fragment is calculated for each charge, stopping at selected max ion charge and associated with the selected retention time and SWATH windows.
  • If any oxonium ions is selected with Ytype modifications, for each unique combination of precursor sequence, the program would also include the appropriate oxonium ions at fragment number of max Ytype fragment number +1 progressively.
